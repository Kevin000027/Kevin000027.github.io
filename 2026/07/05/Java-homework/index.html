案例三 素数对

两个相差为2的素数称为素数对，如5和7，17和19等，本题目要求找出所有两个数均不大于n的素数对。
【输入】
一个正整数n(1≤n≤10000)。
【输出】
所有小于等于n的素数对。每对素数对输出一行，中间用单个空格隔开。若没有找到任何素数对，输出empty。

案例五 打印购物小票

  1. 案例描述
    当顾客在超市购物结账时，收银员会提供一张购物小票，上面会列出顾客购买的商品数量、单价、小计和总计等购物信息，以供顾客核对。本案例要求编写一个模拟打印购物小票的程序，收银员输入顾客购买商品的种类数后，依次输入商品名称、单价和数量。输入完成后打印购物小票，包括商品的名称、单价、数量、小计和总计，小计是指某种商品一共所花金额。
  2. 案例分析
    由案例描述和图1展示的效果图可知，商品包含多个信息，对此可以将商品看作一个对象，通过不同的属性存储对应的商品信息。基于此下面分析案例的实现思路。
    (1) 定义商品类。定义一个商品类用于存储商品的信息，类中定义商品名称、商品单价、购买数量和小计属性，并提供对应的setter和getter方法，其中小计是通过商品单价、购买数量计算获得，对此小计只需提供getter方法，getter方法返回计算结果。
    (2) 定义购物小票类。定义一个购物小票类，在该类中实现打印购物小票的功能。收银员需要提前输入购买商品的种类数，并且购物小票中需要购买商品的种类数显示对应个数的商品信息，对此可以根据输入商品的种类数创建一个商品类型的数组，存储所购买商品的信息。
    (3) 定义保存商品信息的方法。案例需要依次输入所有商品的信息，可以在购物小票类中定义一个保存商品信息的方法，通过循环创建商品对象，初始化商品对象后将其存入数组。
    (4) 定义计算商品总计的方法。购物小票中最后还要显示商品的总计价格，对此可以在购物小票类中定义一个计算商品总计的方法，在该方法中遍历数组，累加所有商品的小计，得到总计价格。
    (5) 定义打印小票的方法。由于小票中的商品信息是规律排列的，可以在购物小票类中定义一个方法根据指定格式输出数组中商品信息和总计价格。

案例六 猜数字游戏

  1. 案例描述
    Java中可以使用Random类来生成随机数，它提供了 nextInt()和nextDouble()等方法来生成随机整数和随机浮点数，示例代码如下。
    Random random = new Random(); //创建一个Random对象
    int randomNumber = random.nextInt(100)+1; //生成一个随机整数
    上述代码中使用Random类生成了一个1~100之间的随机整数，并将其赋值给变量randomNumber。Ramdom类的具体用法将会在后续章节中详细讲解。
    下面需要编写一个程序，生成一个1到100之间的随机数，并让用户猜测该数是多少。根据用户的输入，给出相应的提示，例如，“猜大了”、“猜小了”或“猜对了”。若未猜对，则继续猜，直到用户猜对数字，程序结束。猜数字游戏的效果图如图1所示。
  2. 案例分析
    (1) 从案例描述中可知，首先需要生成一个1到100之间的随机数，对此可以使用Random类生成一个随机数。
    (2) 程序需要将用户输入猜测的数，对应可以使用Scanner类获取用户输入的数。
    (3) 由于不清楚用户需要多少次才能猜中，对此可以使用while语句进行循环判断，当前猜中之后结束循环。

案例九 复数相加

编写一个完整的Java Application 程序使用复数类Complex验证两个复数 1+2i 和3+4i 相加产生一个新的复数 4+6i 。
复数类Complex必须满足如下要求：
(1) 复数类Complex 的属性有：
realPart : int型，代表复数的实数部分
imaginPart : int型，代表复数的虚数部分
(2) 复数类Complex 的方法有：
Complex( ) : 构造函数，将复数的实部和虚部都置0
Complex( int r , int i ) : 构造函数，形参 r 为实部的初值，i为虚部的初值。
Complex complexAdd(Complex a) : 将当前复数对象与形参复数对象相加，所得的结果仍是一个复数值，返回给此方法的调用者。
String toString( ) : 把当前复数对象的实部、虚部组合成 a+bi 的字符串形式，其中a 和 b分别为实部和虚部的数据。
