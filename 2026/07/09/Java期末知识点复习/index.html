面向对象（上）

面向对象的三大特性

封装、继承、多态

类与对象

成员变量：定义在类中、方法外，有默认初始值（如int默认0，String默认null）；
成员方法：定义在类中，无static修饰（实例方法），必须先new出一个对象，通过对象调用。
静态方法：定义在类中，不用new对象，直接 类名.方法名()调用.静态方法中不能使用this关键字（this代表当前对象，静态方法不依赖对象）.静态方法中不能访问实例成员（非静态变量、非静态方法）。
1. 引用变量：存的不是对象本身，只是对象在堆内存的地址，比如代码里 Staff staff ，staff就是引用变量。
2. 值传递场景：int这类基础变量直接存数字，传参只复制数字。
3. 对象的引用传递：把这个引用变量里的地址复制一份传给方法，两个引用都指向同一个实体，改属性自然会同步变化，这就是引用传递的本质。

封装

为什么需要封装？
隐藏属性，暴露方法，通过方法控制属性的访问（如赋值时校验合法性），保护数据完整性。
访问控制修饰符（封装的基础）

封装后的类通常遵循JavaBean规范，便于框架使用和代码统一：
1. 成员变量私有化（private）；
2. 提供公共的getter和setter方法；
通过方法访问属性：外部通过对象名.getXXX()获取属性，对象名.setXXX(值)设置属性
3. 提供无参构造方法（默认存在，若定义有参构造需显式声明）；
4. 可选：重写toString()方法（便于输出对象信息）。

易错点

  1. 构造方法命名错误：将构造方法名写成小写（如student()），导致被识别为普通方法；
  2. 忘记显式声明无参构造：定义有参构造后，默认无参构造失效，若需用无参构造必须显式声明；
  3. static修饰局部变量：静态成员只能修饰类的成员（变量、方法、代码块），不能修饰局部变量；
  4. setter方法校验逻辑缺失：封装时未对属性值进行校验，导致数据不安全；
  5. 混淆静态与实例成员的调用方式：用对象调用静态成员（语法允许，但不推荐），用类名调用实例成员（语法报错）。

继承

继承的概念与特性
• 核心概念：继承是类与类之间的关系，子类（Subclass）继承父类（Superclass）的非私有属性和方法，同时可添加自身特有属性和方法，实现代码复用与扩展。
• 语法格式：class 子类名 extends 父类名 { // 子类特有属性与方法 }
• 核心特性：
￮ 单继承：Java中类只能直接继承一个父类（避免多继承的冲突问题）；
￮ 多层继承：子类可作为其他类的父类，形成继承链（如A→B→C）；
￮ 继承传递性：子类继承父类的属性和方法，也继承父类的父类的非私有属性和方法；
￮ 子类不能继承父类的私有成员（private修饰），但可通过父类的公共方法间接访问。

方法重写（Override）

• 核心概念：子类中定义与父类方法名、参数列表、返回值类型（或子类类型）完全相同的方法，覆盖父类方法的实现，体现子类的特有行为。
• 重写规则：
a. 方法名、参数列表、返回值类型必须与父类一致（返回值可协变：子类返回值是父类返回值的子类）；
b. 子类重写方法的访问权限不能小于父类方法（如父类是protected，子类可改为public，不能改为private）；
c. 父类静态方法不能被重写（静态方法属于类，子类可定义同名静态方法，但属于隐藏，非重写）；
d. 父类私有方法不能被重写（子类无法继承）；
e. 子类重写方法不能抛出比父类方法更多的 checked 异常。

super关键字

• 核心作用：指代父类对象，用于访问父类的成员变量、成员方法、构造方法，解决子类与父类成员同名的冲突。
• 常见用法：
a. 访问父类成员变量：super.父类变量名；
b. 调用父类成员方法：super.父类方法名(参数)；
c. 调用父类构造方法：super(参数)（必须在子类构造方法第一行）。

多态

同一行为作用于不同对象，产生不同的执行结果（运行时多态），即“父类引用指向子类对象，调用方法时表现出子类特性”。
• 实现条件：
a. 继承：子类继承父类（或实现接口）；
b. 重写：子类重写父类的方法（或实现接口的抽象方法）；
c. 向上转型：父类引用变量指向子类对象（父类类型 变量名 = new 子类类型();）。
• 代码案例：多态的实现（动物行为）

对象类型转换
多态中父类引用只能访问父类定义的成员，无法访问子类特有成员，需通过类型转换实现：
• 向上转型（自动转换）：父类类型 变量名 = new 子类类型();，无需显式转换，是多态的基础；
• 向下转型（强制转换）：子类类型 变量名 = (子类类型) 父类引用;，需显式转换，用于访问子类特有成员，转换前需用instanceof判断类型，避免类型转换异常（ClassCastException）。

异常

a. Error（错误）：系统级错误（如内存溢出），程序无法处理，需修改代码或环境；
b. Exception（异常）：程序级异常，可通过代码处理，分为：
▪ Checked异常（编译时异常）：编译时必须处理（如IOException），否则编译报错；
▪ Unchecked异常（运行时异常）：编译时无需处理（如NullPointerException、ArithmeticException），运行时触发。

Math类和Random类

Math类的random()方法仅能生成0.0~1.0的随机double，Random类支持生成多种类型的随机数（int、long、boolean等），功能更强大。

正则表达式

正则表达式是用于匹配、查找、替换字符串的模式串，结合Java的Pattern（编译正则表达式）和Matcher（执行匹配操作）类，可实现复杂的字符串校验与处理。
核心语法（常用）
• 字符匹配：
￮ 普通字符：直接匹配（如”a”匹配字符”a”）；
￮ 转义字符：\d匹配数字（0-9）、\D匹配非数字、\w匹配字母/数字/下划线、\W匹配非字母/数字/下划线、\s匹配空白字符（空格、制表符等）、\S匹配非空白字符；
￮ 范围匹配：[a-z]匹配小写字母、[A-Z]匹配大写字母、[0-9]匹配数字、[a-zA-Z0-9]匹配字母数字；
• 数量限定：
￮ *：匹配前面的字符0次或多次；
￮ +：匹配前面的字符1次或多次；
￮ ?：匹配前面的字符0次或1次；
￮ {n}：匹配前面的字符恰好n次；
￮ {n,}：匹配前面的字符至少n次；
￮ {n,m}：匹配前面的字符n到m次；
• 边界匹配：
￮ ^：匹配字符串开头；
￮ $：匹配字符串结尾；
• 逻辑匹配：
￮ |：逻辑或（如”a|b”匹配”a”或”b”）；
￮ ()：分组（如”(ab)+”匹配”ab”、”abab”等）。

集合和泛型

Java集合框架核心分为两大体系：
• Collection接口：存储单个元素的集合，核心子接口为List（有序可重复）、Set（无序不可重复）；
• Map接口：存储键值对（key-value）的集合，key不可重复，value可重复。

Collection
├─ List（有序、可重复）
│ ├─ ArrayList（数组实现，查询高效）
│ └─ LinkedList（链表实现，增删高效）
└─ Set（无序、不可重复）
├─ HashSet（哈希表实现，无序，查询快）
└─ TreeSet（红黑树实现，有序，自动排序）

Collection接口核心方法（List与Set的公共方法）
• 添加元素：boolean add(E e)（添加单个元素）、boolean addAll(Collection c)（添加集合中所有元素）； • 删除元素：boolean remove(Object o)（删除单个元素）、boolean removeAll(Collection c)（删除集合中所有匹配元素）、void clear()（清空集合）；
• 查找判断：boolean contains(Object o)（判断是否包含元素）、boolean isEmpty()（判断集合是否为空）、int size()（返回集合元素个数）；
• 遍历转换：Iterator iterator()（获取迭代器）、Object[] toArray()（转为数组）。

Map
├─ HashMap（哈希表实现，无序，查询快）
└─ TreeMap（红黑树实现，有序，按key排序）
• 核心方法：
￮ 添加/修改：V put(K key, V value)（key不存在则添加，存在则修改value）；
￮ 删除：V remove(Object key)（通过key删除键值对）、void clear()（清空）；
￮ 查找：V get(Object key)（通过key获取value，不存在返回null）、boolean containsKey(Object key)（判断key是否存在）、boolean containsValue(Object value)（判断value是否存在）；
￮ 遍历：Set keySet()（获取所有key的集合）、Collection values()（获取所有value的集合）、Set<Map.Entry<K,V>> entrySet()（获取键值对集合，推荐遍历方式）。

I/O

IO流用于完成程序和文件间的数据读写，分为字节流、字符流两大分支。

字节流

1. 顶层父类

  • 输入流： InputStream （读取字节数据）
  • 输出流： OutputStream （写出字节数据）
    2. 适用场景：处理音频、视频、图片、压缩包等二进制文件
    3. 配套案例：音频加密与解密，依靠字节流直接操作文件原始字节。

字符流

1. 顶层父类

  • 输入流： Reader （读取文本字符）
  • 输出流： Writer （写出文本字符）
    2. 适用场景：只处理 .txt 等纯文本文件，自动适配编码，不会出现乱码

核心区分
1. 字节流：按单个字节读写，所有文件通用，二进制文件首选；
2. 字符流：按单个字符读写，仅支持文本文件，解决文字编码乱码问题。
